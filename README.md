# GitHub Pages Hub

## 概要・説明
**GitHub Pages Hub (GPHub)** は、散らばりがちな複数の「GitHub Pages」のURLを一元管理・整理するためのWebアプリケーションです。

データベースやバックエンドサーバーを必要とせず、すべてブラウザのローカル環境（IndexedDB）で動作します。登録したURLからGitHub APIを介してリポジトリのタイトルや説明文を自動取得し、見やすいカード形式でリスト化します。検索、お気に入り、タグ・メモ機能による整理はもちろん、Web Workerを用いた非同期のデータ同期や、万が一のための自動バックアップ機能も備えた高機能なPWA（Progressive Web App）です。

## 主な機能一覧

* **GitHub Pagesの一元管理**: URL、カスタムタイトル、メモ（タグ）を登録し、カードUIで分かりやすく管理。
* **メタデータの自動取得**: Web Workerを利用し、バックグラウンドでGitHub APIと通信してタイトルや説明文、最終更新日を自動取得。
* **一括同期・スリープ防止**: 登録した全ページを一括で最新状態に更新。Wake Lock APIにより、同期処理中の画面スリープを防止します。
* **検索・お気に入り（フィルタリング）**: URL、タイトル、メモを対象にしたインクリメンタルサーチと、スター（お気に入り）による絞り込み。
* **自動バックアップと復元**: データを保存するたびに直近5世代までの状態を自動バックアップし、いつでも過去の状態に復元可能。
* **インポート / エクスポート**: 全データをJSON形式でファイルとしてダウンロード・読み込み可能。
* **API制限緩和設定**: GitHub PAT (Personal Access Token) を設定することで、APIリクエストの制限を回避。
* **PWA対応**: マニフェストファイルを動的生成し、スマートフォンやPCにネイティブアプリのようにインストール可能。
* **下書き保存**: ページ追加途中の入力内容をローカルストレージに自動保存。

## 使用技術・ライブラリ

* **フロントエンド**: HTML5, CSS3, JavaScript (Vanilla JS / フレームワーク不使用)
* **スタイリング**: Tailwind CSS (CDN経由)
* **アイコン**: Font Awesome 6.4.0 (CDN経由)
* **ストレージ・データ管理**: 
  * IndexedDB (メインデータ・バックアップ保存)
  * LocalStorage (設定・下書き保存)
* **主要なWebブラウザAPI**:
  * Web Workers API (非同期通信・メインスレッドのブロック防止)
  * Screen Wake Lock API (バックグラウンド同期中のスリープ防止)
  * File API (データのインポート/エクスポート)

## セットアップ・使い方

### セットアップ
本アプリケーションはビルド環境やサーバーを必要としません。
1. 本HTMLファイルをダウンロード、またはリポジトリをクローンします。
2. ダウンロードした `index.html` を直接Webブラウザ（Chrome, Edge, Safari推奨）で開くだけですぐに使用開始できます。
3. （推奨）GitHub PagesやVercel、Netlifyなどの静的ホスティングサービスにデプロイすることで、どこからでもアクセスできるようになります。

### 使い方

1. **ページの追加**
   * 画面下部中央の青い「＋」ボタンをクリックします。
   * 対象のGitHub PagesのURL（例: `https://username.github.io/repo/`）を入力して保存します。
   * 自動的にバックグラウンド通信が走り、リポジトリ情報が取得されます。
2. **ページの管理・検索**
   * 画面上部の検索バーから、URLやメモの内容で目的のページを即座に探せます。
   * 各カードの「☆」アイコンをクリックすると、お気に入りリストに登録されます。
3. **データの一括更新**
   * 画面下部の「一括更新」ボタンを押すと、登録された全ページのメタデータ（説明文や更新日時）を一斉に再取得します。
4. **設定とバックアップ**
   * ヘッダー右上（歯車アイコン）から「設定モーダル」を開くことができます。
   * GitHub APIのレート制限に引っかかる場合は、ご自身の「GitHub PAT」を発行し、ここに登録してください。
   * 機種変更やブラウザ移行の際は、この画面から「エクスポート（JSONダウンロード）」し、新しい環境で「インポート」してください。
5. **誤操作からの復元**
   * 誤ってページを削除してしまった場合は、画面下部の「履歴/復元」ボタンから過去のバックアップポイントを選択し、データを復旧できます。
